FPGA Real-Time Vision Tracking System

Built a full imaging pipeline on a Spartan-7 FPGA using a CMV300 CMOS sensor. Custom Verilog modules handle SPI camera configuration, synchronized pixel sampling, 32-bit packing, FIFO buffering, and high-throughput USB transfer via Opal Kelly BlockPipeOut. Frame-aligned padding meets USB block constraints and prevents FIFO overflow.

A Python host application receives frames, reconstructs grayscale images, performs target detection, and commands a motor driver for real-time closed-loop camera panning. System throughput is bottlenecked by motor response time rather than USB bandwidth.

Verilog / RTL Spartan-7 FPGA SPI USB (Opal Kelly) Python Image Processing Motor Control

RISC-V Unix-like OS Kernel

Built a Unix-like OS kernel for RISC-V from scratch in C. Implemented Sv39 virtual memory (page-table walks, demand paging, TLB flush via sfence.vma), cooperative and preemptive threading with condition variables, fork/exec/wait process lifecycle, kernel pipes with correct blocking/EOF semantics, a block-based filesystem with inode/bitmap allocators, and a user-space shell supporting I/O redirection and pipelines. Debugged across assembly/C ABI boundaries using QEMU and GDB.

C RISC-V Assembly QEMU GDB Virtual Memory Concurrency Filesystems

Duck Hunt on FPGA

NES-style Duck Hunt built on a Spartan-7 FPGA using SystemVerilog and C. Implemented a modular VGA/HDMI video pipeline, sprite rendering engine, palette-indexed ROM graphics, and FSM-based game-state logic. LFSR-based pseudo-random duck movement, pixel-level color compositing with priority and transparency, and all animations synchronized to game state. USB mouse input handled via MicroBlaze soft processor over AXI-Lite — fully playable in real time.

SystemVerilog C Spartan-7 FPGA VGA / HDMI MicroBlaze AXI-Lite FSM

Forage — Smart Pantry App

Cross-platform mobile app (React Native + Expo + TypeScript) with offline-first SQLite storage. Barcode scanning via expo-camera auto-fills product details from the Open Food Facts API. Items are sorted by predicted expiration with storage-aware shelf-life rules. Architected with Expo Router screens and modular /lib layers for DB, date utilities, and API client.

In development — roadmap includes OCR expiry scanning, LLM-powered recipe suggestions, and receipt import.
